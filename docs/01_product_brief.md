# More Rent — product brief

## 1. Суть проекта
Нужно разработать современный сайт автопроката для бренда **«Море Рент»** в России. Сайт должен выглядеть дорого, уверенно и технологично, на уровне сильных международных брендов, но при этом быть понятным, быстрым и конверсионным.

Главная задача сайта — не просто показать автомобили, а **довести пользователя до отправки заявки на аренду**.

## 2. Бизнес-цель
Сайт должен:
- вызывать доверие за первые 5–10 секунд
- показывать регион работы и основной формат услуги
- демонстрировать автомобили, тарифы и преимущества
- собирать заявки без обязательной регистрации
- передавать заявки в **GreenCRM** и создавать там сделку

## 3. Главный пользовательский сценарий
Основной flow:
1. Пользователь заходит на сайт
2. Сразу понимает, чем занимается компания, где работает и почему ей можно доверять
3. Переходит в каталог / подбор авто
4. Смотрит карточки и характеристики автомобилей
5. Выбирает автомобиль
6. Переходит к заказу
7. Заполняет короткую форму
8. Видит понятный экран успеха
9. Менеджер получает заявку, а пользователь ожидает обратной связи

## 4. Каналы прихода пользователей
По брифу основные источники:
- прямые заходы
- поисковые системы: Google, Яндекс
- Avito

Это значит, что сайт должен быть одновременно:
- сильным по SEO
- быстрым и понятным с первого экрана
- готовым к трафику с коммерческим намерением

## 5. Что важно для клиента
Из брифа:
- доверие к компании
- понятность пути
- рабочая форма без перегруза
- нормальная мобильная версия
- каталог машин с ценами и характеристиками

## 6. Что раздражает пользователя сейчас
Согласно брифу:
- недостаточный авторитет сайта
- высокая цена относительно конкурентов

Значит сайт должен компенсировать это через:
- премиальный внешний вид
- сильные блоки доверия
- прозрачные тарифы
- акцент на ценности, а не только цене
- визуальную демонстрацию качества сервиса

## 7. Устройства
Трафик делится примерно 50/50:
- 50% desktop
- 50% mobile

Следовательно:
- mobile UX критичен
- desktop нельзя делать вторичным
- дизайн должен быть адаптивным, а не просто «ужатым»

## 8. Тональность бренда
По брифу тон сайта должен быть **официальный**, но для достижения уровня сильного бренда нужен микс:
- официальная структура сообщений
- чистая, дорогая визуальная подача
- уверенные короткие тексты
- без дешевых маркетинговых штампов

## 9. MVP scope
Обязательный MVP:
- главная
- список авто с ценами и характеристиками
- форма заказа
- о компании
- контакты
- интеграция заявок с GreenCRM

Можно вынести во 2 этап:
- путеводитель по региону
- SEO-посадочные под НЧ запросы
- акции
- отзывы как отдельный развитый раздел
- доп. оборудование
- продвинутый калькулятор

## 10. Что не нужно в первой версии
- личный кабинет
- статус заявки на сайте
- загрузка файлов до общения с менеджером
- встроенный чат
- календарь со слотами
- сложные многошаговые анкеты
