# More Rent — brand, UX, UI, motion

## 1. Визуальное направление
Сайт должен передавать ощущение:
- надежности
- движения
- свободы
- премиальности
- морского и южного характера бренда

Не делать «типичный прокат авто из шаблона». Нужен фирменный, атмосферный, дорогой digital look.

## 2. Цветовая система
Основной цвет бренда: **синий**.

Рекомендуемая палитра:
- Primary Blue: `#0B5FFF`
- Deep Ocean: `#073B8C`
- Dark Navy: `#071C35`
- Sky Accent: `#66B3FF`
- Soft Ice: `#EAF4FF`
- White: `#FFFFFF`
- Graphite Text: `#101828`
- Muted Gray: `#667085`

Использование:
- основной CTA: Primary Blue
- hover/active: Deep Ocean
- темные секции hero/footer: Dark Navy
- светлые фоновые зоны: White / Soft Ice
- акценты, разделители, glow: Sky Accent

## 3. Типографика
Рекомендуемая пара:
- headings: `Manrope` / `Sora`
- body: `Inter`

Требования:
- короткие крупные заголовки
- большой контраст размеров
- не использовать тяжелые длинные абзацы
- ключевые цифры и CTA должны читаться мгновенно

## 4. Общий UI-подход
Нужен интерфейс класса premium tech brand:
- много воздуха
- крупные блоки
- понятная иерархия
- карточки автомобилей с clean layout
- мягкие тени
- glass / blur использовать умеренно
- не перегружать градиентами

## 5. 3D и motion language
3D и анимации должны усиливать восприятие, а не мешать.

### Что использовать
- 3D parallax в hero
- объемные световые блики
- плавные reveal-анимации на скролле
- микроанимации карточек авто
- мягкое смещение слоев при наведении
- динамичные переходы между страницами
- subtle motion backgrounds

### Что не использовать
- тяжелые WebGL-сцены на каждой секции
- агрессивные вращения
- анимации, тормозящие mobile
- навязчивые эффекты ради эффекта

### Рекомендуемый стек для motion
- Framer Motion
- GSAP — точечно, где реально нужно
- CSS transform / opacity для большинства эффектов
- React Three Fiber — только если делается легкая hero-сцена или showcase

## 6. Кастомный курсор
Нужен кастомный курсор, но он не должен раздражать.

### Поведение курсора
- базовый вид: маленький светящийся круг / точка
- внешний ring при движении
- hover state на кнопках: увеличение + подпись типа `Смотреть`, `Выбрать`, `Открыть`
- на draggable / gallery зонах: смена состояния
- на мобильных устройствах курсор отключается полностью

### Ограничения
- не ломать accessibility
- не скрывать системный курсор там, где это неудобно
- не использовать курсор в формах и текстовых полях, если это мешает UX
- предусмотреть fallback

## 7. UX-принципы
- пользователь за 5 секунд понимает, что здесь можно арендовать авто
- путь до заявки максимально короткий
- ключевые CTA повторяются на странице
- фильтрация авто должна быть простой
- на мобильном все действия доступны большим пальцем
- формы короткие, понятные, без лишних полей
- каждая страница заканчивается следующим действием

## 8. Hero section
Hero должен решать 3 задачи:
1. мгновенно объяснить предложение
2. вызвать доверие
3. перевести в подбор/заказ

### Состав hero
- сильный headline
- короткий подзаголовок
- 1–2 CTA
- визуал автомобиля / кинематографичный motion visual
- микро-блок доверия: регион, условия, опыт, скорость ответа

### Пример композиции
Слева:
- headline
- subheadline
- CTA buttons
- badges / metrics

Справа:
- крупный hero-car visual
- light / shadow / 3D layers
- мягкий motion

## 9. Карточки автомобилей
Каждая карточка должна включать:
- фото
- название модели
- класс / тип
- цена от
- краткие характеристики
- CTA `Подробнее` и/или `Арендовать`

Характеристики лучше показывать через 3–5 коротких бейджей.

## 10. Блоки доверия
Обязательные trust blocks:
- почему выбирают нас
- прозрачные условия
- реальные преимущества сервиса
- география / регион работы
- отзывы
- юридическая прозрачность
- понятные условия возврата и оплаты

## 11. Адаптивность
Нужны отдельные продуманные решения для:
- hero mobile
- каталог mobile
- sticky CTA на мобильном
- форма бронирования mobile
- footer / contacts mobile

Не просто уменьшать desktop layout — проектировать mobile отдельно.
